Bloom Filters

A bloom filter is a space-efficient probabilistic data structure used to test whether an element is a member of a set. Possible results:
  • Possibly in the set
  • Definetly not in the set

In Ethereum, each block has a bloom filter, `logsBloom`, which contains the address of any logging contract and its indexed fields from the logs generated

Every bloom filter is a 2048 bit string (512 hex chars). See example of `logsBloom` in block `17932776`:

        "0x6931135cc5a6515f5e59469cc06c26f1f4885aa6800c7f88281fe42bf22dcf982020a9ddc24f4235f491734ced1873a476b124c79caefb064c35193ee1682bbcae46c888d359283ca98a601ada6c0b2ca05d2442be62ce0b0287e64caa35cc05770f251da7120c3bb8e08870ea48be5bca1aa88c145b45b89e24e55cd06ef2be0f9820c61c835921b15c08640713d44e582a6855f9e5963c2a211cefa5fa901ccbed1164cea763e3215e67c4bd880c466c634c664170c50740c9f9aaa0288a7ae5b2148314ceda62a50b0a26b729ba833875ca487f7efa1e1815429e10e0af6857baa95caddacd915fd51be8a1bd9c8e9526f2797baab97c4125a8669890fc31"

Every log that is emitted in a transaction has 0-4 topics. Each topic will set 3 bits to `1` based on the hash of the topic.

It takes the first (least-significant) 11 bits of the KECCAK-2561 of the transaction address and indexed topics.
